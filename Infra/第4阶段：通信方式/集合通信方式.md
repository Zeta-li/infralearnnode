# 集合通信方式

| 操作              | 输入（每节点） | 输出（每节点）   | 通信量             | 主要用途   |
| ----------------- | -------------- | ---------------- | ------------------ | ---------- |
| **AllReduce**     | 相同形状的张量 | 归约后的相同张量 | 2×(N-1)/N × 数据量 | 梯度同步   |
| **AllGather**     | 不同/相同张量  | 拼接后的完整张量 | (N-1) × 数据量     | 激活值聚合 |
| **ReduceScatter** | 相同形状的张量 | 归约后的分块张量 | (N-1)/N × 数据量   | 优化器分片 |
| **Broadcast**     | 根节点有数据   | 所有节点获得副本 | (N-1) × 数据量     | 权重初始化 |
| **Reduce**        | 相同形状的张量 | 仅根节点获得结果 | (N-1)/N × 数据量   | 指标收集   |

**all2all**,向不同的节点发送不同数据

先列切，再行切，直接all reduce

ALLReduce：

中心化：**Reduce + Broadcast**根节点接受再发送。根节点通信量（通信总量）：2(*N*−1)*M*，其他节点：2M

ring：**ReduceScatter**+**AllGather**每个节点处理一部分。每个节点通信量：2(N−1)M/N，（通信总量）：2(N−1)M



| 维度     | All-to-All(all2all)                    | Dispatch（分发）       | Combine（合并）             |
| -------- | -------------------------------------- | ---------------------- | --------------------------- |
| 层级     | 底层通信原语（通用）                   | MoE 上层业务算子       | MoE 上层业务算子            |
| 本质     | 数据交换工具                           | 路由分发阶段           | 结果回收融合阶段            |
| 出现次数 | Dispatch、Combine 各调用 1 次，共 2 次 | MoE 每层前半段 1 次    | MoE 每层后半段 1 次         |
| 数据流向 | 双向交换，无固定方向                   | Token 源卡 → 专家卡    | 专家卡 → Token 源卡（逆向） |
| 额外计算 | 纯数据拷贝，无计算                     | 仅打包、排序 token     | 通信后加权求和专家输出      |
| 输入内容 | 任意分片张量                           | 原始输入 token、路由表 | 专家计算后的输出            |
| 依赖关系 | Dispatch/Combine 依赖它                | 内部封装一次 all2all   | 内部封装一次 all2all        |
