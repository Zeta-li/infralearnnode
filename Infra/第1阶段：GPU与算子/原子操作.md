### 为什么要存在原子操作

在并行计算中，多个线程同时更新同一数据可能导致数据竞争，引发结果不一致问题。为解决这一挑战，CUDA 提供了原子函数，借助硬件支持确保操作的原子性，保障共享数据在并发访问时的正确性与高效性。

当多个线程同时访问并写入同一个显存地址时，最终结果将是不确定的。由于缺乏同步机制，写入结果完全取决于线程执行的时序。**在最坏情况下，N个线程都执行了加1操作，但最终内存中的值可能仅增加1，而非预期的N。**这就是典型的多线程编程中的竞态条件问题。

具体过程如下：

1. 多个线程同时读取相同的初始值
2. 每个线程在自己的寄存器中对该值执行加一操作
3. 多个线程尝试将修改后的值写回同一内存位置

### 解决方案

- 为了解决上述问题，例如在直方图计算中我们需要对所有数据进行累加得到最终结果时，CUDA 提供了一套完整的原子操作（atomic operations）函数，用于有效避免多个线程并发访问共享内存时引发的竞态条件。
- 原子操作通过CUDA API提供，确保对全局内存中某个地址的“读-修改-写”操作是不可分割的，即整个操作过程不会被其他线程中断。换句话说，**在执行原子操作期间，目标内存地址会被硬件锁定，其他尝试访问同一地址的线程将被阻塞，直到当前操作完成并释放锁。**访问该位置的其他线程通常会被阻塞或保留在队列中，直到轮到它们，线程串行执行原子操作。

| 操作     | 函数名                        | 说明                                 |
| -------- | ----------------------------- | ------------------------------------ |
| 加法     | atomicAdd(addr, val)          | 加法                                 |
| 减法     | atomicSub(addr, val)          | 减法                                 |
| 最大值   | atomicMax(addr, val)          | 返回两个值的最大值                   |
| 最小值   | atomicMin(addr, val)          | 返回两个值的最小值                   |
| 与       | atomicAnd(addr, val)          | 按位与                               |
| 或       | atomicOr(addr, val)           | 按位或                               |
| 异或     | atomicXor(addr, val)          | 按位异或                             |
| 交换     | atomicExch(addr, val)         | 设置新值并返回旧值                   |
| 比较交换 | atomicCAS(addr, compare, val) | 如果当前值等于 compare，则设置为 val |

如果使用原子操作，会造成串行执行，降低效率，所以：

1. **使用共享显存：**竞争被限制在了线程块（Block）内部。每个Block只有256个线程在竞争自己的`histo_private`。

2. **合并回全局显存：**每个Block完成后，只需向全局显存提交一次（256个Bin）结果。全局显存收到的原子操作总量从N（数据总量）降低到了GridDim × 256。

   ```c++
   __global__ void hist(int8_t *input, int *hist, int n)
   {
       __shared__ int histo_private[256];
   
       // 初始化私有直方图（每个block内清0自己的共享内存空间）
       for (int j = threadIdx.x; j < 256; j += blockDim.x) {
           histo_private[j] = 0;
       }
       __syncthreads();
   
       // 构建私有直方图（每个线程负责n/总线程数）
       int i = threadIdx.x + blockIdx.x * blockDim.x;
       for (int idx = i; idx < n; idx += gridDim.x * blockDim.x)//gridDim.x（block总数） * blockDim.x=总线程数
       {
           int8_t in = input[idx];
           if (in >= 0 && in < 256)
           {
               atomicAdd(&histo_private[in], 1);
           }
       }
       __syncthreads();
   
       // 将私有直方图合并到全局直方图（每个线程把共享内存的内容➕到全局内存中）
       for (int j = threadIdx.x; j < 256; j += blockDim.x) {
           atomicAdd(&hist[j], histo_private[j]);
       }
   ```
